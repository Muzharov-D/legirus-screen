## 1. Что уже сделано (текущее состояние)

### Продукт
**АванDата** — веб-приложение для одного клуба ФК Легирус (СПб, 4 возрастных группы 2010-2013), MVP в production. Раздаётся как PWA, родители ставят на домашний экран.

**Что видит родитель** (`mobile.legirus.sportdata.tech`):
- Лендинг с выбором года рождения ребёнка (2013, 2012, 2011, 2010 — карточки)
- Расписание матчей и тренировок команды (FlashScore-style)
- Турнирная таблица лиги + клубный зачёт + кубковая сетка
- Карточка матча с inline таймлайном событий (голы, замены)
- Push-уведомления о матчах, тренировках, вызовах
- Offline-first (работает на стадионе без связи)

**Что видит тренер** (login):
- Свой календарь, callup-roster (вызвать игроков), тренировки, посещаемость
- Может звать игроков из младших команд (cross-age callups)

**Что видит главный тренер клуба:**
- Все команды клуба, /club с турнирными таблицами и кубками всех возрастов

### Стек
- Backend: Node.js + Express + PostgreSQL (на Render Frankfurt, платный план)
- Frontend: React 18 + Vite + React Router 6 (на Vercel)
- PWA: Service Worker offline-first, Web Push (VAPID)
- Источник данных: **FFSPB API** (Федерация футбола СПб) + scraper SportVisor для match-events
- Кеш: Vercel edge cache `s-maxage=60s SWR=300s` для public API
- Cron-обновление с FFSPB: **каждые 30 минут** (calendar / standings / cup / match-events)

### Архитектурные решения уже заложены под мульти-клубность
- Sprint 4 ТЗ — multi-club design (но пока хостит только Легирус)
- Public-страница `/public/team/:age` уже параметризуется по age, без жёсткой привязки к клубу
- В PG модель данных поддерживает несколько клубов (есть `clubs` таблица)
- FFSPB API отдаёт **данные всех клубов СПб** — мы пока фильтруем только Легирус

### Что НЕ сделано
- UI пока бренд-привязанный к Легирус (логотип, цвета, тексты «ФК Легирус»)
- Нет выбора клуба на лендинге — только команды одного клуба
- Нет поиска по клубам / командам / игрокам
- Нет монетизации
- Нет аналитики использования (DAU, retention)
- Нет соглашения с FFSPB о коммерческом использовании данных

## 2. Звёздочка: расширение в B2C-платформу

**Заказчик готов платить за продукт.** Направление развития: превратить МVP для одного клуба в **публичную B2C-платформу для родителей всех клубов под FFSPB**.

### Масштаб
- ~11 клубов в СПб (по данным club-rank, точное число в `backend/data/standings/_config.json`)
- 4-5 возрастных групп на клуб = 50+ команд
- Тысячи родителей потенциально
- Все нужные данные **уже доступны через FFSPB API** — не нужно договариваться с клубами

### Ключевая идея
Родитель НЕ ходит на сайт ФК Легирус. Родитель ходит на единое приложение, выбирает свой клуб + команду + ребёнка, и видит:
- Расписание матчей и тренировок СВОЕГО ребёнка
- Турнирные таблицы где играет его команда
- Live-обновления счётов матчей
- Push-уведомления о ВАЖНЫХ для него событиях

### Возможные дополнительные фичи (для обсуждения)
- Профиль ребёнка с favorite team
- Уведомления о матчах конкретно его команды (не просто всей)
- Сравнение команд / игроков (если разрешит этика — про детей сложно)
- Календарь для родителя на месяц с матчами и тренировками
- Карта стадионов СПб (у нас уже есть venues.json)
- Лента новостей клубов

## 5. Юридика и этика — обязательно обсудить

### FFSPB как источник данных
- FFSPB — публичная федерация, данные расписания и таблиц публикуются открыто
- Их API — публичный (мы используем без секретов)
- Но **коммерческое использование** — серая зона. Нужно либо:
  - Получить от FFSPB explicit разрешение / партнёрство
  - Или использовать публикуемые данные «как есть» с reference to source

### Дети до 14 лет — особо защищённая категория
- Имена / фотографии детей **нельзя публиковать без явного согласия родителя**
- Сейчас public-страница показывает счета и команды, но НЕ имена игроков и НЕ фото — это правильно
- При расширении на B2C — нужен юрист для согласования tone of voice

### 152-ФЗ (персональные данные)
- Если будем собирать данные родителей (email, телефон, любимая команда ребёнка) — нужна регистрация в Роскомнадзоре как оператор ПД
- Хранилище ПД — на серверах в РФ (сейчас Render Frankfurt — нарушение)
- Это критично перед B2C запуском

## 6. Бизнес-модели — варианты для обсуждения

| Модель | Плюс | Минус |
|---|---|---|
| **Freemium** (базовое бесплатно, премиум 99₽/мес) | Низкий порог входа, монетизирует engaged | Премиум-фичи нужно придумать |
| **Реклама** (спонсоры — экипировка, лагеря) | Без оплаты от пользователя | Нужна аудитория >5К DAU |
| **B2B SaaS клубам** (бренд-страница + аналитика) | Уже частично сделано (см. /club) | Долгий sales-цикл |
| **Маркетплейс** (форма, тренеры, лагеря — комиссия 10%) | Высокий потенциал | Нужны поставщики и юр. база |
| **Подписка для тренеров/клубов** | Профильная аудитория | Малый TAM |
| **Гибрид** (B2C freemium + B2B SaaS) | Двойной поток | Сложнее в управлении |

## 7. Инфраструктурные ограничения сейчас

- **TSPU/Roskomnadzor**: Vercel и Render IP частично заблокированы у мобильных операторов РФ. Сейчас домен `mobile.legirus.sportdata.tech` работает не у всех. Подготовлены конфиги для миграции на VPS / Yandex Cloud (см. `deploy/`)
- **Render free → платный**: уже платный, no cold start
- **FFSPB rate-limit**: ~30 req/час сейчас, в безопасной зоне. При масштабе 1000 родителей — все равно ОК (мы кешируем, родители смотрят cache)
- **Push-подписки в PG**: нет лимита, но при 10К пользователях — нужен мониторинг
