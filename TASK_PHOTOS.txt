ЗАДАЧА: Проставить фото игрокам клуба
======================================

ЦЕЛЬ
----
В системе у игроков команды Легирус 2010 г.р. уже стоят фотографии.
У остальных команд клуба (2009, 2011, 2012, 2013) фото нет — UI показывает
плейсхолдер с инициалами в круге. Надо это исправить.


ИСХОДНЫЕ ДАННЫЕ (за основу взять)
---------------------------------
Файл:    backend/data/players.json
Эталон:  игроки команды legirus-2010 — у всех есть поле "photo" с путём
         вида "/assets/players/p17-turapin.jpg"


ПОШАГОВАЯ ИНСТРУКЦИЯ
--------------------

1) Сделай свежий pull, чтобы работать с актуальным состоянием:

      cd "C:\Users\dmuzharov\Documents\Claude\Projects\Экран Легирус"
      git pull origin main


2) Открой backend/data/players.json. Найди блок игроков 2010 — посмотри
   как у них сделано поле "photo":

      "photo": "/assets/players/p17-turapin.jpg"

   Это путь относительно frontend/public/. То есть реальный файл лежит:

      frontend/public/assets/players/p17-turapin.jpg


3) Открой папку:

      frontend/public/assets/players/

   Там увидишь файлы по схеме:  p{NN}-{lastname-translit}.jpg
   Например: p05-galitsky.jpg, p17-turapin.jpg, p21-bobin.jpg


4) Пройдись по каждому игроку команд legirus-2009, legirus-2011,
   legirus-2012, legirus-2013 (это поле "teamId" в players.json).

   Для каждого игрока:

   а) Возьми его id (поле "id"), например "p33-makarov".

   б) Поищи в папке frontend/public/assets/players/ файл с этим id:
      p33-makarov.jpg / .jpeg / .png

   в) Если файл найден  — пропиши путь в players.json:

         "photo": "/assets/players/p33-makarov.jpg"

      (имя файла точно как лежит в папке, с расширением)

   г) Если файла нет — оставь "photo": null или вообще удали поле "photo"
      у этого игрока. UI покажет плейсхолдер с инициалами — это норма.


5) Если нужно ДОБАВИТЬ новые фото (тебе их прислали в Telegram/почте):

   а) Сожми изображение до 600x600 px (или 800x800), JPEG quality 80.
      Удобно через https://squoosh.app или MS Paint → Resize.

   б) Назови файл строго по id игрока. Например:
         id "p11-ivanov"  →  файл  p11-ivanov.jpg

   в) Положи в frontend/public/assets/players/

   г) И только теперь пропиши путь в players.json (см. пункт 4в).


6) Сохрани файл. Проверь что JSON валиден:

   Самый простой способ — открой players.json в VS Code, он подсветит
   ошибки красным. Или скопируй содержимое в jsonlint.com.

   Если кавычки/запятые посыпались — пользуйся Ctrl+Z и попробуй ещё
   раз более аккуратно.


7) Закоммить и запушь:

      cd "C:\Users\dmuzharov\Documents\Claude\Projects\Экран Легирус"
      git add backend/data/players.json frontend/public/assets/players/
      git commit -m "feat(players): photos for all club teams"
      git push origin main


8) Через 1-2 минуты Vercel пересоберёт фронт. Зайди на сайт под
   тренером, открой /players → раздел РЕЙТИНГ → пройди по списку:
   у всех должны быть реальные фотки вместо инициалов.


ВАЖНЫЕ ПРАВИЛА (НЕ НАРУШАТЬ!)
-----------------------------

* НЕ МЕНЯЙ поле "id" у игроков — это связь с матчами и базой данных.
  Сломаешь id → сломаешь всю статистику игрока во всех матчах.

* НЕ МЕНЯЙ поле "teamId" — это привязка к команде.

* НЕ ПЕРЕИМЕНОВЫВАЙ файлы фото "просто чтобы было красивее".
  Имя файла должно совпадать с id игрока — иначе UI его не найдёт.

* НЕ ДОБАВЛЯЙ фото игроков ДРУГИХ клубов (Пороховчанин, Самсон и т.д.)
  Только наши команды клуба Легирус.

* JPG предпочтительнее PNG — он легче и быстрее грузится. PNG только
  если у фото прозрачный фон (что редко нужно для портретов).

* Если что-то пошло не так — НЕ ПУШЬ кривой JSON. Откати изменения
  через `git checkout backend/data/players.json` и начни сначала.


СПИСОК КОМАНД, КОТОРЫМ НУЖНЫ ФОТО
---------------------------------

* legirus-2009  (12-15 игроков, ageGroup U-18)
* legirus-2011  (10-15 игроков, U-16) — приоритет, есть данные матчей
* legirus-2012  (~15 игроков, U-15)
* legirus-2013  (~15 игроков, U-14)


ВРЕМЯ
-----
Точное количество игроков посмотришь в players.json после фильтра
по teamId. На каждого игрока — 30 секунд работы (если фото уже в папке)
или 2 минуты (если надо ресайзить и положить новое).

Всего на задачу — 1-2 часа максимум.
